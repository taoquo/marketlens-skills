# Trade Setup

Use this file when an event may create a trading window. Keep the output as research guidance, not personalized trading advice.

## Purpose

An event can be important without being attractive to trade. A trade setup requires an identifiable expectation gap, a clear mechanism, a dated window, and a risk/reward read that is not already fully priced.

## Setup Components

| Component | What to state |
|---|---|
| Consensus view | What published estimates, guidance, or common market narrative imply |
| Variant view | What could be different from consensus and why |
| Market-implied expectation | What price action, valuation move, implied move, IV, short interest, or flows appear to price |
| Implied move | Options-implied move or volatility context when available; mark unavailable if not sourced |
| Upside scenario | What must happen for a positive thesis read-through |
| Downside scenario | What would impair the thesis or create de-risking |
| Risk/reward read | Whether upside surprise appears large enough relative to downside and implied move |
| Invalidating evidence | Data that would make the setup event watch, monitor closely, or avoid |
| Review window | When to judge the event after release, guidance, or first revision cycle |

## Interpretation Rules

- If expectation gap is unclear, use `event watch`.
- If implied move is high, the event needs a larger surprise to create favorable price reaction.
- If crowding is extreme, use `monitor closely` or `event watch` unless the event directly challenges the crowded thesis.
- If downside scenario is hard to bound, do not call the setup high priority.
- If the event has no dated review window, treat it as strategic monitoring rather than a tactical setup.
